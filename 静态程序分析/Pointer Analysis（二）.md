- **定义**：它是指针 $p$ 在程序执行过程中，**可能**指向的所有堆对象的集合

**$P(O)$**：如果 $O = \{o_1, o_2\}$，那么 $P(O) = \{\emptyset, \{o_1\}, \{o_2\}, \{o_1, o_2\}\}$。

> 为什么指针传播的过程中是通过o_i.f 而不是 x.f 来进行传播

简单来说：**`x.f` 是不稳定的（会变），而 $o_i$ 是稳定的（唯一标识）。**
```
1: x = new T(); // 记为 o1
2: y = x;       // y 也指向 o1
3: x.f = z;     // 存储操作
```
如果你通过 `x.f` 来传播 `z` 的指向，那么当你查询 `y.f` 指向谁时，分析器会发现它只知道 `x.f` 的信息，而不知道 `y.f`。

**正确的逻辑（基于 $o_i.f$）：**
1. 因为 $pt(x) = \{o_1\}$，所以 `x.f = z` 实际上是把 `z` 的指向传给了 **$o_1.f$**。
2. 因为 $pt(y) = \{o_1\}$，当你分析 `y.f` 时，分析器会去查 **$o_1.f$** 的指向。
3. 这样，`x` 和 `y` 就能通过 $o_1$ 这个“中介”成功共享信息，识别出它们是**别名**。
